# UART 帧格式详解
Universal Asynchronous Receiver Transmitter (UART)即通用异步收发器，是一种通用的串行、异步通信总线。该总线有两条数据线，可以实现全双工的发送和接收。在嵌入式系统中常用于主机与辅助设备之间的通信。

## 波特率
波特率用于描述UART通信时通信速度bps(bit per second),即每秒钟传送的bitt数。波特率一般取9600bps，即每秒钟传送9600个bit。

## UART帧格式
UART帧格式一般包含以下几种：
1. 8N1 - 8bit数据，无校验位，一位停止位。
2. 8N2 - 8bit数据，奇偶校验位，一位停止位。
3. 8E1 - 8bit数据，偶校验位，一位停止位。
4. 8E2 - 8bit数据，偶校验位，两位停止位。
5. 8O1 - 8bit数据，奇校验位，一位停止位。
6. 8O2 - 8bit数据，奇校验位，两位停止位。

以8N1为例，UART帧格式为：
起始位 | 数据位（LSB...MSB）               | 奇偶校验位（可选） | 停止位
  0   | D0  D1  D2  D3  D4  D5  D6  D7  |   P             |   1

偶校验的规则是：包括校验位在内，整个字节（数据位+校验位）中“1”的总个数为偶数。
奇校验的规则是：包括校验位在内，整个字节（数据位+校验位）中“1”的总个数为奇数。
校验位只能校验但不能纠错.

UART总线上空闲时为高电平。传输时需要置低电平。
停止为可以为1，1.5，2bit。停止位为高电平。

串口通信一次发送一个字节，即8bit数据。主要用于防止误差累计。
异步通信协议是指收发双方时钟不同步。SPI和IIC都是同步总线协议。

## UART时钟与数据传输时钟同步
UART没有时钟线，双方各自使用独立的内部时钟.
传输1 bit需要固定的位时间（由波特率决定，例如波特率为9600bps，1bit为1/9600s，即104us）。
接收端为了准确采样，通常会用一个更高频率的本地时钟（如16倍波特率时钟）来对每个bit进行过采样。

同步串行通信如SPI，IIC有独立的时钟线SCLK。发送器和接收器共用同一个时钟信号。通常每个时钟周期传输1个bit（例如在时钟的上升沿和下降沿采样数据）。

## UART控制器
一般情况下，SoC中都会集成UART控制器，用于UART通信。UART控制器一般包含以下功能： 1. UART控制器的初始化。 2. UART控制器的配置。 3. UART控制器的接收。 4. UART控制器的发送。
我们使用UART进行通信时只需对其内部的相关寄存器进行设置即可。

# RS232， RS485协议原理

## UART协议的缺点
- UART只是对信号的时序进行了定义，而未定义接口的电气特性.比如，规定了高电平代表1，低电平代表0，但是没有规定多少伏特时高电平，多少伏特是低电平.
- UART通信时一般直接使用处理器使用的电平，称为TTL电平，但不同的处理器使用的电平存在差异，所以不同的处理器使用UART通信时一般不能直接相连。
- 抗干扰能力差。UART一般直接使用TTL信号来表示0或1，但TTL信号的抗干扰能力较差，数据在传输过程中很容易出错。故其通信距离也非常短，一般只能用于一个电路板上的两个不同芯片之间的通信。
- UART没有规定不同器件连接时连接器的标准，所以不同器件之间通过UART通信时连接很不方便

## RS232协议
- RS232协议是在1970年由美国电子工业协会（EIA）联合贝尔系统、调制解调器厂家，计算机终端生产厂家共同制定的用于串行通讯的标准。
- 该标准规定采用一个标准的连接器，标准中对连接器的每个引脚的作用加以规定，还对信号的电平加以规定.
- 该标准采用一个25引脚的DB-25连接器，该连接器的每个引脚都规定了其作用，以及该引脚所使用的电平。后来IBM的PC机将RS232简化成了DB-9连接器，后来成为事实标准；现在工业控制的RS232采用DB-9连接器，并规定其引脚的电平为5V。接口一般只使用RXD、TXD和GND三个引脚，RXD为接收，TXD为发送，GND为地。

[DB9 connectors](../illustrations/DB9_connectors.png)
[DB9 pins definition](../illustrations/DB9_pins.webp)

- 改标准规定逻辑“1”的电平为-5v到-15v，逻辑“0”的电平为+5v到+15v。选用该电气标准的目的在于提高抗干扰能力，增大通信距离，其传送距离一般可达15m。

## 电平转换
虽然很多处理器中都会集成UART控制器，但处理器产生的信号一般都是TTL信号并不是符合RS232信号，因此还需要在处理器外部添加电路对信号电平进行转换。对于绝大多数嵌入式开发场景，推荐使用 ‌MAX3232 (3.3V系统)‌ 或 ‌MAX232 (5V系统)‌ 芯片，配合4颗0.1μF电容构成最小系统。这种方案成熟、稳定且易于布线，是TTL与RS232电平转换的标准实践 。‌‌

## 编程
RS232 串口编程和UART编程一样。RS232只是在电气层进行了规定。

## RS232存在的问题
- 接口的信号电平值较高，易损坏接口电路的芯片，又因为与TTL电平不兼容，所以需要使用电平转换芯片才能与TTL电路连接。
- 通信速度较低，传输距离较短(15m).
- 易产生共模干扰，抗噪声干扰性弱。

# RS485 协议
该标准由电信行业协会和电子工业联盟定义；使用该标准的通信网络能在**远距离条件下(可达1500m)**及**电子噪声大的环境**下有效传输信号；该标准允许连接多个收发器，即具有多站能力，这样可以利用单一的RS485接口实现多个设备之间的通信，方便地建立起一个设备网络.

[RS485 示意图](../illustrations/RS485.png)
RS485总线上可以接最多32个设备，每个设备都有两个引脚，一个用于接收数据，一个用于发送数据。

## 信号
RS485标准规定采用差分信号进行数据传输，两线间的电压差为+2v到+6v表示逻辑"1",两线间的电压差为-2v到-6v表示逻辑"0"，两条线描述一个信号；使用差分信号能有效地减少噪声信号的干扰，延长通信距离，RS485的通信距离可以达到1500m。RS485接口信号的电平比RS232降低了，所以不易损坏接口电路的芯片，且该电平与TTL电平兼容，可方便地与TTL信号连接。
RS485的两条线一般做成双绞线，使得两条线收到的干扰相同。

## 接口
RS485采用两线制，这种接线方式为总线式拓扑结构，在同一总线上可以同时存在多个节点.
因为采用两线制，数据的发送和接收都要使用这对差分信号线，发送和接收不能同时进行，所以只能采用半双工方式工作，编程时也需要加以处理。
可以仿照IIC的方式管理总线，即总线总线控制，数据控制，数据传输。主机和从机不是硬件区分，而是软件区分。

## 电平转换
虽然很多处理器中都会集成UART控制器，但处理器产生的信号一般都是TTL信号并不是符合RS485信号，因此还需要在处理器外部添加电路对信号电平进行转换。常用的芯片有MAX485，MAX487等。

## RS485的优势
- 接口的信号电平值较低，不易损坏接口电路的芯片，且与TTL电平兼容，可方便地与TTL电路连接。
- 通信速度快。抗噪声干扰性强。传输距离较远(1500m左右)
- 可实现多节点组网。

# IIC总线
IIC（Inter-Integrated Circuit)总线是Philips公司在1980年代初退出的一种**串行、半双工**总线，主要用于**近距离、低速**的芯片之间的通信。IIC总线有**2根双向的信号线，即SCL(Serial Clock Line)和SDA(Serial Data Line)**。时钟线SCL用于通信双方的时钟同步；数据线SDA用于收发数据。

IIC总线是一种**多主机总线**，连接在总线上的器件分为主机和从机。主机有权发起和结束一次通信，而从机只能被主机呼叫。当总线上有多个主机同时启用总线时，IIC总线会自动进行仲裁，并选择一个主机作为主主机。每个连接到IIC总线上的期间都有一个唯一的地址（7bit），且每个期间可以作为主机也可以作为从机（同一时刻只能有一个主机），总线上的器件的增加和删除不影响其他期间正常工作。IIC总线在通信时总线上发送数据的期间为发送器，接收数据的器件为接收器。

[IIC示意图](./illustrates/iic.webp)

## IIC总线通信过程
- 1. 主机发送起始信号启用IIC总线；
- 2. 主机发送一个字节数据指明从机地址和后续字节的发送方向；
  字节中bit1-bit7为从机地址，bit0为发送方向，0为主机发送数据（主机->从机），1为主机接收数据（从机->主机）；
- 3. 被寻址的从机发送应答信号回应主机；
- 4. 发送器发送一个字节数据
- 5. 接收器发送应答信号回应发送器；
- ...循环上述第4，5步
- n. 通信完成后，主机发送停止信号释放总线。

## IIC总线的寻址方式
- IIC总线上传送的数据是广义的，既包括地址，又包含真正的数据。
- 主机在发送起始信号后必须先发送一个字节的数据，该数据的高7位为从机地址；最低位将自己定义为发送器或接收器。

## IIC总线上的信号
### 起始信号和停止信号
- 当SCL和SDA同时处于高电平时，规定总线为空闲状态。
- SCL为高电平时，SDA由高变低表示起始信号。
- SCL为高电平时，SDA由低变高表示停止信号。
- 起始信号和停止信号都是由主机发出，起始信号产生后，总线处于占用状态，停止信号产生后，总线空闲。

[IIC起始信号和停止信号](./illustrates/iic_signals.png)

### 字节传送与应答
- IIC总线通信时每个字节为8bit长度，数据传送时，先传送最高位，后传送低位，发送器发送完一个字节数据后必须发送1位应答位来回应，即一帧共有9bit。
- 数据线SDA上，发送数据时，逻辑“1”为高电平，逻辑“0”为低电平。应答信号为接收器相应的信号，即接收器收到数据后，向发送器发送一个逻辑“0”（低电平）表示应答。如果接收器没有收到数据或不想应答，则向发送器发送一个逻辑“1”（高电平）表示没有应答。

### 同步信号
通信的时候要解决的2个问题：
- 在通信开始前，必须保证接收器知道发送器什么时候开始发送数据。在IIC协议下是通过主机发送起始信号实现。
- 接收器要知道收到的信号是一个逻辑“1/0”还是n个逻辑“1/0”，
IIC总线在进行数据传输是，时钟线SCL为低电平期间，发送器向数据线上发送一位数据，在此期间数据线上的信号允许发生变化；时钟线SCL为高电平期间，接收器从数据线上读取一位数据，在此期间数据线上的信号不允许发生变化，必须保持稳定。
SCL的作用就是告诉发送器何时发数据；告诉接收器何时收数据。又因为SCL即连接在发送器上又连接在接收器上，它们共享同一个时钟，所以IIC可以发送和接收任意长的数据。
一个字节（8bit）的发送要经过SCL上9次高低变化.

# SPI总线
SPI（Serial Peripheral Interface）是串行外设接口的缩写，SPI是一种高速、全双工、同步的串行通信总线。SPI采用主从方式工作，一般有一个主设备和一个或多个从设备；SPI需要**至少**4根线，分别是MISO（主设备输入从设备输出），MOSI（主设备输出从设备输入），SCLK（时钟）和CS（片选）。SPI使用引脚较少且布线方便，所以越来越多的芯片集成了这种通信协议。

## 寻址方式
SPI总线使用片选CS（chip select）作为寻址方式。当主设备要和某个从设备进行通信时，主设备需要先向对应的从设备的片选线上发送使能信号（高电平或低电平，根据从机而定，在电路图中，若CS上有一个横杠，说明低电平有效，若没有横杠，说明高电平有效）表示选中该设备。

## 通信过程
SPI总线在进行数据传送时，先传送高位，后传送低位；数据线为高电平表示逻辑“1”，为低电平表示逻辑“0”；一个字节传送完成后无需应答即可开始下一个字节的传送；SPI总线采用同步方式工作，时钟线在上升沿或下降沿时发送器向数据线上发送数据，在紧接着的下降沿或上升沿时接收器向数据线上读取数据，完成一位数据传送，八个时钟周期即可完成一个字节数据的传送。

SPI总线没有起始信号也没有停止信号，接收器收到数据也没有应答信号。

## 极性和相位
SPI总线有四种不同的工作模式，取决于极性（CPOL）和相位（CPHA）两个因素。
CPOL表示SCLK空闲时的状态：
- CPOL=0时，SCLK空闲时为低电平；
- CPOL=1时，SCLK空闲时为高电平。
CPHA表示SCLK上升沿或下降沿时数据线状态：
- CPHA=0时，下降沿读数据，上升沿发数据；
- CPHA=1时，下降沿发数据，上升沿读数据。

对一个特定的从设备，一般在出厂时就会将其设计为某种特定的工作模式。在使用该设备时，必须保证主设备的工作模式和该从设备保持一致，否则无法进行通信。
所以，一般需要对主设备的CPOL和CPHA进行配置。

## SPI和IIC的异同
### 相同点
- 均采用串行、同步的方式进行通信；
- 均采用TTL电平，传输距离和应用场景类似；
- 均采用主从方式工作。

### 不同点
- IIC为半双工，SPI为全双工；
- IIC总线采用8bit进行数据传输，而SPI总线采用任意长度的数据进行传输；
- IIC有应答机制，SPI无应答机制；
- IIC总线有起始信号和停止信号，SPI无起始信号和停止信号。
- IIC通过总线广播从机地址来寻址，SPI通过向对应从机的片选线上发送使能信号来寻址；
- IIC的时钟极性和时钟相位固定，SPI的时钟极性和时钟相位可配置。
